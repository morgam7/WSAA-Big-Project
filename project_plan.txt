

At minimum, 
a Flask REST API, one database table, and a web interface using AJAX to do CRUD.

Project can be:

Lichen sightings tracker

A user can:

create a lichen sighting
read/view all sightings
update a sighting
delete a sighting

Each sighting could have:

lichen name
description
location name
latitude
longitude
maybe image path later

That already satisfies the minimum project.

So  simplest acceptable version, project is:

One table: lichens

id
name
description
location_name
latitude
longitude

Flask API

GET /lichens
POST /lichens
PUT /lichens/<id>
DELETE /lichens/<id>

Frontend

HTML/CSS/JS page
uses AJAX/fetch calls to Flask
lets user add, view, edit, delete lichen records

That is enough for the minimum.

Then the extra marks:

CSS

“More than one database table”

add users

search by county

all good extras later

So the best strategy is:

Version 1 for the assignment

AJAX frontend

host it online

map of sightings

one table
